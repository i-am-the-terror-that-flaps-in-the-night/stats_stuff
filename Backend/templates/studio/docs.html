How this project is built

One FastAPI server does everything: the dashboard, the JSON stats API, the static assets and these pages. The pages call the API's own cached functions directly — same process, same memory — so no request ever leaves the server to draw a table.

The three layers

LayerFileJob
Stats engineBackend/engine.py Pure statistics on a dataframe. Imports only numpy, pandas and scipy — it knows nothing about the web, so everything above it can reuse it and nothing above it can break it.
APIBackend/app.py Wraps the engine in HTTP endpoints and caches every answer, so repeat questions are free.
SiteBackend/studio.py These pages. FastAPI routes that render Jinja2 templates and call the API's cached functions directly — same process, same memory.

How missing data is handled

Missing values are dropped, never invented. A cell that can't be read as a number stays missing rather than being replaced with a zero or an average, so a gap in the data never quietly turns into a data point that pulls the mean around.
